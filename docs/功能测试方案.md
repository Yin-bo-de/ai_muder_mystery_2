# AI案件解谜应用 - 功能测试方案

## 1. 测试策略概述

### 1.1 测试目标
验证AI案件解谜应用的核心功能符合产品需求，确保：
- 案件生成逻辑完整、自洽
- 线索搜查、对话质询、指认凶手等核心玩法正常
- AI角色行为符合人设一致性要求
- 端到端用户体验流畅

### 1.2 测试范围

#### 接口测试（API Testing）
- 游戏管理接口：创建案件、获取状态、勘查、解密、指认
- AI交互接口：发送消息、切换模式、控场指令
- 用户管理接口：用户信息、历史记录

#### 端到端测试（E2E Testing）
- 完整游戏流程：开局 → 探案 → 结案
- 核心玩法场景：现场勘查、单独审讯、全体质询、指认凶手
- 关键交互：@嫌疑人功能、线索关联、矛盾检测提示

### 1.3 测试环境
- 后端：`http://localhost:8000`
- 前端：`http://localhost:5173`
- 测试框架：pytest（接口）、agent-browser（E2E）

**前置条件：**

## 6. 缺陷等级定义

| 等级 | 描述 | 示例 |
|------|------|------|
| P0 - 阻塞 | 无法进行测试，核心功能不可用 | 无法创建案件、页面白屏 |
| P1 - 严重 | 核心功能异常，主要流程无法完成 | 无法指认凶手、线索不保存 |
| P2 - 一般 | 次要功能异常，不影响主要流程 | 打字机效果失效、UI样式问题 |
| P3 - 轻微 | 体验优化建议 | 文案错别字、按钮位置不理想 |
